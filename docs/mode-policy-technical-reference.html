Mode Policy Technical Reference

Contract for how readings choose relationship mode, register band, intake fields, forbidden assumptions, and downstream source-pack behavior.

Contract Shape

{
  mode: "solo | married | partner-relationship | general-synastry | family | business",
  register: "l1_l3 | l4_l5",
  requiredContextFields: string[],
  intakeQuestions: string[],
  requiredAnchors: string[],
  forbiddenFrames: string[],
  sourceRules: string[]
}

Consumers

Reading Runner

Validates mode context before LLM calls and injects policy/context into the system prompt.

Asset Factory

Writes policy and answered context into the NotebookLM source pack.

Audit Chain

Verifies source packs do not contradict deterministic facts or allowed assumptions.

Grounding and Inference Dependencies

NVIDIA Embeddings

BGE-M3 1024-dimensional embeddings turn fact/perspective queries into vectors for retrieval.

Cloudflare Vectorize

witness-wisdom-corpus stores wisdom passages and subject engine data.

LLM Providers

NVIDIA, OpenRouter, and OpenAI sit behind provider factory/inference adapters.

Dual-Agent Interaction

Aletheios

Receives structural/engine context and speaks as the observing witness.

Pichet

Receives body/state context and speaks as the embodied walker.

Anti-Dependency

Every mode policy should preserve user sovereignty: the system succeeds when the user needs it less.
